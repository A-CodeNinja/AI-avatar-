# AI头像小程序总执行方案（2026-03-06）

## 1. 方向判断：你现在的方向是对的

你的核心方向是可行的：
1. 用户上传照片 -> AI生成头像
2. 叠加头像框/贴纸/民族风元素
3. 下载和分享成品
4. 用积分体系和头像墙形成复访

建议的商业闭环：
1. 免费体验：新用户送积分 + 每日签到 + 看广告获取积分
2. 付费转化：会员包（月卡）或积分包（后续再开）
3. 裂变增长：分享作品、邀请好友、头像墙榜单
4. 内容驱动：每周上新素材 + 周榜高赞作品

结论：主方向正确，优先做「可用、稳定、持续上新」比一次做很多功能更重要。

## 4. 豆包API在微信开发者工具中的配置步骤（详细）

### 4.1 获取豆包（火山引擎 Ark）参数

你需要：
1. `AI_API_KEY`
2. `AI_API_URL`（图生图推荐 `https://ark.cn-beijing.volces.com/api/v3/images/edits`）
3. `AI_MODEL`（图生图推荐 `doubao-seedream-3-0-i2i-250328`）
4. `AI_TYPE` 固定填 `doubao`

### 4.2 在云函数目录配置

编辑文件：`cloudfunctions/generateAvatar/config.json`

示例：

```json
{
  "env": "你的云环境ID",
  "envVariables": {
    "AI_API_KEY": "你的ARK_API_KEY",
    "AI_API_URL": "https://ark.cn-beijing.volces.com/api/v3/images/edits",
    "AI_MODEL": "doubao-seedream-3-0-i2i-250328",
    "AI_TYPE": "doubao"
  }
}
```

### 4.3 微信开发者工具部署

1. 打开云开发面板，确认环境 ID 与 `env` 一致
2. 右键 `cloudfunctions/generateAvatar` -> 上传并部署：云端安装依赖
3. 对新增函数同样部署：
   1. `updateUserProfile`
   2. `submitFeedback`
   3. `toggleAvatarLike`
   4. `weeklyCleanupGallery`
   5. `initDatabase`
4. 手动调用一次 `initDatabase`

### 4.4 功能联调顺序

1. 先登录（授权）
2. 再上传图片
3. 调用生成
4. 查看积分扣减和记录
5. 去头像墙点赞
6. 提交一条反馈

## 8. 备案、认证、ICP：你接下来要做什么

说明：以下为通用流程建议，具体以微信公众平台和当地主管部门最新要求为准。

### 8.1 微信小程序认证

准备材料：
1. 主体信息（个人/个体户/企业）
2. 管理员身份信息
3. 对公信息（企业主体）
4. 认证费用与发票信息

建议：如果要长期做商业化，尽量用企业主体。

### 8.2 小程序备案

核心是小程序内容、域名、服务器信息对应一致。

准备项：
1. 主体证件
2. 负责人信息
3. 服务内容描述
4. 服务器接入信息

### 8.3 ICP相关

常见情况：
1. 使用云开发且不涉及独立网站域名时，按平台要求走小程序备案流程
2. 若后续接入独立官网或H5，通常需要对应 ICP 备案
3. 若涉及经营性互联网服务，可能涉及增值电信许可（按地区和业务判定）

建议：上线前找本地合规服务商做一次清单核验，避免走弯路。

通用提示词结构：
1. 风格：民族/情侣/游戏/节日
2. 元素：纹样、配色、饰物
